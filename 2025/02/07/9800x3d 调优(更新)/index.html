小参参考

可以适当加大soc至1.2v

  • 可以的话 GDM context restore Power Down 全给关了
  • GDM = on makes your system more stable but only allows even tCL. Solution: Either lower the clock speed or enable GDM.
    • . Gear down mode is the one allows the ram to run off its internally generated ½ rate clock for latching on the command or address buses. so if you do 1T with Gear Down Mode, it will at .5T off sync, making it like a 1.5T, with the trade of latency. for latency, 1T > 2T > 1T GDM, for performance 1T > 1T GDM > or = 2T.
    • I prefer to use 2T to gain better stability and better latency since no off sync command rate is down, with sacrificing a little performance (not that much).
  • 华硕gdm 是内存界面 哪个buf
    • ADDR_CMD_Mode in DRAM Timing Control
    • Buf means disabled

tPHYRDL

  • 顺便说一下，我再次停用了 SWAP APU 历史记录。您在读取过程中获得了 400 MB/秒的速度，但在复制过程中损失了 2,000 MB/秒的速度。至少对于我来说，这是不值得的。
  • 关于同步 tPHYRDL（调整 CLDO VDDp 或其他电压甚至电阻）的主题，当涉及到两个存储体上 tPHYRDL 的同步训练时，它对我没有任何帮助。
  • 最终对我有帮助的是，他以相同的方式在两个工作台上训练 tPHYRDL，该选项是：ARdPtrInitVal P0 Control 为 1。您可以尝试 0、1 或 2。你瞧，他终于不再对 A2/B2 进行不同的训练了。
  • 只是一个提示，我认为这对某些人来说是不同步的。它不会提高性能，但至少它展示了一种在 9000 系列上实际修复该问题的方法！ ARdPtrInitVal P0 控制选项仅在 9000 系列的 BIOS 中可用！

  - 终于搞定了同步问题,我选的是2 —-RdPtrInitVal P0 Control 为 2
  -

  • 对于游戏 PC，您不需要 VBS、Credential Guard、HVCI、Spectre Mitigations，也不需要 IOMMU。这就是为什么我有
    1. AMD SVM、IOMMU 在 BIOS 中禁用
    2. 通过 InSpectre 禁用 Spectre 缓解措施
    3. 高性能电源计划处于活动状态
